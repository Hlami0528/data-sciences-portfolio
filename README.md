## 📊 Exploratory Data Analysis

### Project 01 — Survival Patterns Aboard the Titanic
***What made the difference between life and death on the Titanic?***
This project digs into one of history's most studied datasets to uncover the demographic and socioeconomic factors that determined passenger survival. Through systematic visual exploration, the analysis reveals how class, gender, and age intersected in the chaos of the disaster 

📎 [Project Folder](./Exploratory-Data-Analysis/Project_01_Titanic_EDA)

**Skills:** `Python` · `Pandas` · `NumPy` · `Seaborn` · `Matplotlib` · `Jupyter` · `Data Analysis` · `Data Visualisation`

### Project 02 — Decoding Emergency Incidents Across Cape Town
***Where does Cape Town's emergency response system face the greatest pressure?***
Using open data published by the City of Cape Town, this project analyses thousands of emergency incident records to identify spatial hotspots, seasonal trends, and the most prevalent categories of urban crises. The goal: empower city planners and emergency services with a clearer picture of where and when incidents cluster across the metro.

📎 [Project Folder](./Exploratory-Data-Analysis/Project_02_Cape_Town_Incidents_EDA)

**Skills:** `Python` · `Pandas` · `Seaborn` · `Matplotlib` · `Geospatial Analysis` · `Jupyter` · `Data Visualisation`



### Project 03 — Predicting Fetal Health from Cardiotocogram Data
***Can a non-invasive scan help prevent maternal and child mortality?***

With 295,000 maternal deaths and millions of at-risk pregnancies annually, early detection is critical. This project explores 2,126 cardiotocogram (CTG) exam records which is classified by expert obstetricians as Normal, Suspect, or Pathological. The goal: to identify which fetal health indicators carry the most diagnostic weight.

### Project 04 — Code Language Classification (R vs Python)
***Can a model read code and tell you which language it's written in?***

RStudio developers needed a smarter way to catch mismatched code chunks in R Markdown documents — before irrelevant errors confuse users. This project builds a text classification model trained on 549 real R and Python scripts. The goal: to enable RStudio developers to automatically detect and flag language mismatches in real time.

**Skills:** `R` · `Text Classification` · `NLP` · `Machine Learning` · `tidyverse` · `R Markdown` · `Feature Engineering`· `tidytext` · `Feature Engineering `· `caret`


### Project 05 — Forecasting Daily Bicycle Demand in London
***How do you keep 20 million bike transactions from becoming a redistribution nightmare?***

London's bike-sharing network lives or dies on supply-demand balance. Using over 20 million transactions spanning 2017–2018, this regression project builds a model to predict daily bicycle demand at stations across the city. The goal: to enable smarter, data-driven fleet redistribution and reducing the frustration of empty docks and full stations.

📎 [View Project Report](#)

**Skills:** `Python` · `Scikit-learn` · `Pandas` · `Regression Modelling` · `Feature Engineering` · `Time Series` · `Matplotlib`

### Project 06 — Chronic Kidney Disease Prediction
***Early detection could change outcomes for millions of patients.***

Chronic Kidney Disease (CKD) affects hundreds of millions worldwide, yet early-stage diagnosis remains inconsistent. Using clinical data from 400 patients across 25 medical features, this project builds and evaluates classification models with a focus on robust preprocessing and comparing Naïve Bayes imputation against baseline mode/mean strategies to handle missing clinical data before modelling.

📎 [View Project Report](#)

**Skills:** `R` · `Naïve Bayes` · `Classification` · `Data Imputation` · `tidyverse` · `Healthcare ML` · `Model Evaluation`



### Project 07 — German Credit Risk Classification
***Who's a good borrower? A data-driven answer using k-NN and Naïve Bayes.***

This project applies k-Nearest Neighbours and Naïve Bayes classifiers to the German Credit dataset, evaluating which customers represent acceptable lending risk. Beyond initial accuracy, the analysis confronts overfitting head-on — using 10-fold cross-validation to expose how well each model generalises beyond its training data, and what that means for real-world credit decisions.

📎 [View Project Report](#)

**Skills:** `R` · `k-NN` · `Naïve Bayes` · `Cross-Validation` · `Feature Correlation` · `tidyverse` · `Model Evaluation`



## 🗣️ Natural Language Processing


### Project 08 — Discovering Infrastructure Themes in South African News
***What are journalists telling us about South Africa's crumbling infrastructure — and are we listening?***

The South African Institute of Civil Engineering (SAICE) compiles Infrastructure Report Cards to assess the state of the country's roads, water systems, and public works. But official data alone is often incomplete. This project mines online news articles to discover recurring infrastructure themes through unsupervised clustering — creating an alternative, scalable intelligence source to support future infrastructure assessments.
